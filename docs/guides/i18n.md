# i18n — Internationalization Guide

Convergio ships with English defaults for all UI strings. Downstream
projects can override any string via YAML config or React context.

## Quick start — YAML

Add a `locale:` section to `maranello.yaml` (or `convergio.yaml`):

```yaml
locale:
  shell:
    skipToContent: "Saltar al contenido principal"
    dashboard: "Panel"
  a11yFab:
    accessibilitySettings: "Configuración de accesibilidad"
    textSize: "Tamaño de texto"
  dataTable:
    loading: "Cargando..."
    filterPlaceholder: "Filtrar..."
    previousPage: "Página anterior"
    nextPage: "Página siguiente"
```

Only override what you need — English defaults fill the rest.

## Quick start — React context

Wrap your app with `MnLocaleProvider`:

```tsx
import { MnLocaleProvider } from "@/lib/i18n";

// From your own i18n system
const messages = locale === "es" ? esOverrides : undefined;

<MnLocaleProvider messages={messages}>
  <App />
</MnLocaleProvider>
```

The dashboard layout already wraps children with `MnLocaleProvider`
and passes YAML overrides automatically.

## How it works

```
YAML locale: section
        │
        ▼
MnLocaleProvider (React context)
        │
        ▼
useLocale("namespace") → merged labels
```

Priority: **YAML overrides > English defaults**.

## Using in components

```tsx
import { useLocale } from "@/lib/i18n";

function MyComponent() {
  const t = useLocale("dataTable");
  return <span>{t.loading}</span>;
}
```

For server components (outside the provider):

```tsx
import { resolveLocale } from "@/lib/i18n";

function ServerPage() {
  const t = resolveLocale("errorPage");
  return <h1>{t.somethingWentWrong}</h1>;
}
```

## Adding i18n to a new component

1. Define a new interface in `src/lib/i18n/types.ts`
2. Add English defaults in the matching `src/lib/i18n/defaults/*.ts` file
3. Add the namespace to `LocaleMessages` in `types.ts`
4. Wire defaults in `src/lib/i18n/defaults/index.ts`
5. Use `useLocale("yourNamespace")` in the component
